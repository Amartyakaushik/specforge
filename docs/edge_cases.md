# Edge Cases

| ID | Prompt | Type | Expected Behavior | Handling Strategy |
|----|--------|------|-------------------|-------------------|
| e1 | "Build me an app" | Vague | Generate a generic CRUD application with Users entity, dashboard, and basic auth | Analysis stage infers a default app type (task manager). Fills in entities (User, Task), pages (dashboard, list, create/edit), and navigation. Documents all assumptions in output metadata. |
| e2 | "Blog platform where authors can't edit posts but editors can, and authors have full edit access" | Conflicting | Resolve permission conflict, produce consistent RBAC | Detect contradictory permission statements during analysis. Resolve toward the more permissive interpretation (authors can edit own posts, editors can edit all). Log conflict resolution in `assumptions` field. |
| e3 | "thing for managing stuff" | Vague | Produce a minimal but functional item management app | Map "thing" → application, "stuff" → generic Item entity with name, description, status fields. Generate CRUD pages, basic nav. Minimal viable config that passes all validators. |
| e4 | "Social media platform with live streaming, AI content moderation, blockchain-based payments, and AR filters" | Overspecified | Scope to feasible subset, flag unsupported features | Analysis stage categorizes features as supported (social: posts, comments, users) and unsupported (streaming, blockchain, AR). Generate config for supported subset. Return `unsupportedFeatures` array in response with explanation. |
| e5 | "A todo app that's also an ERP system and a CRM and uses machine learning for predictions" | Conflicting scope | Identify primary type, include secondary features where feasible | Classify primary intent as todo/task management. Include lightweight CRM features (contacts entity). Drop ERP and ML as out of scope. Document scope decisions. Entity count stays under 10. |
| e6 | "Admin dashboard where admins cannot access the admin pages" | Logical contradiction | Resolve toward sensible default | Detect logical contradiction (admin role denied admin pages). Resolve: admins get full access to admin pages. Log the contradiction and resolution. Auth validator confirms role-route consistency post-resolution. |
| e7 | "Task tracker" | Incomplete | Generate a complete task tracking application with sensible defaults | Expand to: Task entity (title, description, status, priority, due date, assignee), User entity, dashboard with task stats widget, list/create/edit pages, status-based filtering. Fill all gaps with conventional defaults. |
| e8 | "Project management tool with 50 different entity types including..." (followed by extensive list) | Overspecified | Cap entities at reasonable limit, warn user | Cap at 20 entities. Prioritize by dependency order (core entities first: User, Project, Task; then secondary). Return `truncated: true` with list of dropped entities. All included entities are fully valid. |
| e9 | "" (empty string) | Adversarial | Reject with clear error message | Fail fast at input validation before pipeline execution. Return 400 with `{ error: "Prompt is required", code: "EMPTY_PROMPT" }`. No tokens consumed, no pipeline stages executed. |
| e10 | "Real-time app with zero latency, infinite storage, and offline-first that's always online" | Impossible constraints | Ignore contradictions, build feasible subset | Strip impossible qualifiers during analysis (zero latency, infinite storage, always-online + offline-first). Generate a standard real-time application config with WebSocket support notation and local storage caching. Log dropped constraints. |
